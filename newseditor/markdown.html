Markdown

Markdown syntax is commonly used in web-based content management systems. By abbreviating some HTML, markdown makes it easier to read and proof the input content. For markup not covered by markdown syntax, like tables, images, iframes and videos, use regular HTML. The text below this point can be cut and pasted into an input field to see the result on the subscriber site.

Headers use one to six hash characters at the start of the line, corresponding to header levels one through six. For example:

#This is an H1

##This is an H2

###This is an H3

Paragraphs, which are separated with a blank line, use two returns at the end.

New lines  
without a blank line  
in between them  
use two spaces  
at the end of the line.

*Italic* text is surrounded by single asterisks, **bold** text is surrounded by double asterisks.

For links, the anchor text is delimited by [square brackets]. To create an inline link, use a set of regular parentheses immediately after the link text's closing square bracket. Inside the parentheses, put the URL where you want the link to point.

[This link](http://example.net/) is for a web page.

This link, [John Doe](mailto:john.doe@example.net), is for an e-mail address.

Like tables, images must be entered with regular HTML. Images are sized in the style sheet, not with specific pixel dimensions in the HTML. Specific to Online News Site, a few style classes exist for sizing images. The class "w" is for images that use the full page width. The lead classes are for smaller images with text wrapped to the right of the image. The image size gets smaller as the lead-class number increases. The smallest, lead4 images, are thumbnails. The additional image class b puts a border around the image. Here are examples beginning with full-width class w.

<img class="w b" src="images/logo.png" alt="" />

<img class="lead b" src="images/logo.png" alt="" />Lead

<img class="lead2 b" src="images/logo.png" alt="" />Lead2

<img class="lead3" src="images/logo.png" alt="" />Lead3

<img class="lead4" src="images/logo.png" alt="" />Lead4

Markdown supports blockquotes, unordered (bulleted) and ordered (numbered) lists.

Blockquotes use a greater-than sign followed by a space:

> Indented text.

Unordered lists use a dash followed by a space:

- Red
- Green
- Blue

Ordered lists use a number followed by a period and a space:

1. Bird
2. McHale
3. Parish

It's important to note it's possible to trigger an ordered list by accident, by writing something like this:

1986. What a great season.

To avoid the result, escape the period with two preceding backslashes:

1986\\. What a great season.

Markdown provides backslash escapes for the characters below, which all can have an interpreted meaning within markdown depending upon where they occur. See the full markdown documentation for more information. When one of the characters is interpreted but is supposed to be as typed, then add two backslashes before the character.

\ backslash
` backtick
* asterisk
_ underscore
{} curly braces
[] square brackets
() parentheses
> greater than
# hash mark
+ plus sign
- minus sign (hyphen)
. dot
! exclamation mark
